1. Setup Firebase

MKTURK uses Google Firebase for database, storage and backend service needs. Therefore it is imperative that Firebase is setup correctly. In this section, we will guide you through the steps of setting up Firebase projects, Firebase CLI as well gsutil and gcloud command line interfaces.

Create a Firebase Project

Navigate to Firebase, then click 'Add project'. After following the prompt, you will have created a new project. You can enable or disable Google Analytics but for simplicity, you can disable it now and always enable it later.

Add a Webapp

Now that you've created a Firebase project, you need to add a Webapp since MKTURK is a web-based software. Click on the Web icon (pointed by the red arrow in the image below) on the Firebase project landing page. Once you click, you will be guided through another set of configuration prompt. Name your app as 'mkturk' and check the box to set up Firebase Hosting.

Install Firebase CLI & Google Cloud SDK

Before continuing to install Firebase CLI, make sure that you have nvm, node and npm installed as you will need a local Node environment on your computer to install and use Firebase, gsutil and gcloud CLIs. The following guide is based on MacOS and Linux.

The recommended way to install Node environment is through the use of nvm ('Node Version Manager'). You can think of nvm as a sort of conda for Node which allows you to install and setup multiple versions of Node depending on the usage.

To check if you have a working installation of nvm, open up your terminal and run:
$ command -v nvm

which should output nvm if you have nvm.

If you do not have nvm, please visit nvm for MacOS/Linux and follow their installation steps.

Now that you have installed nvm, install the latest Node v10 by running
$ nvm install 10
followed by
$ nvm use 10
At this point you have successfully installed nvm, node and npm

Installing Firebase CLI
$ npm install -g firebase-tools

Import Template Database & Storage

MKTURK takes advantage of the different Google Firebase and Cloud database infrastructure. Data that need to be queried are saved in Firestore while data dump operations usually takes place in Google Cloud Storage. In order to get MKTURK working correctly, it is crucial that the databases are initialized correctly.

2. Setup MKTURK

Text about setting up and installing mkturk

Clone MKTURK Repository

To clone Issalab's distribution of MKTURK, run:
$ git clone https://github.com/issalab/mkturk.git

Install Modules

Simply running MKTURK does not additional module installations. In fact, all data manipulations can be done through the native Google Firebase/Cloud consoles. However, MKTURK includes a webapp called MKFILES that centralizes all data viewing/editing/updating operations. To build this webapp, run the following sequence of commands on terminal:
$ cd ~/mkturk/src/mkfiles
$ npm install
$ npx webpack
At this point, navigate to ~/mkturk/public/mkfiles and check to see that mkfiles.js was added.
